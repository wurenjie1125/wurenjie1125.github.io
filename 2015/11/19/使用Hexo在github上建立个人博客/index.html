引言：

建立个人博客，在IT界是一件非常普遍且有意义的事情，毕竟好记性不如烂笔头。既能备忘，又能分享。
如果是将博客在自己的站点上部署，就更棒了。首先，你可能会接触到如何管理后台（虽然Hexo帮你处理了），
其次，你可能会学习服务器知识（如果你从AWS之类购买主机的话需要自己部署，然而github帮你也处理了），
最后，自己选择域名、样式、控制整个站点，不是一个全站工程师的梦想么？
这个博客也是基于Hexo和github建立的。

1、Hexo

Hexo是一个用node.js开发的一个博客框架。博客内容的存储全是通过Markdown的标记形式存储的.md文件。
开发过程十分迅速，可用模板丰富。

1.1 安装Hexo

首先命令行用npm全局安装hexo

$ npm install hexo-cli -g

(若提示没有npm命令，请自行安装node)

这时候我们已经可以使用Hexo命令了。以为hexo已经被全局安装。

在想要建工程文件夹的目录里初始化hexo项目

$ hexo init <文件夹名称>

进入此目录

$ cd <文件夹名称>

安装包依赖

$ npm install

安装之后会在此目录下生成 node_modules文件夹，里面包含hexo各种依赖包

之后生成/更新（每次更改代码，都要进行下面两边操作重新生成网页，重启服务器）

$ hexo generate (可以缩写 hexo g)

启动服务器，系统会在默认4000端口开启，可以在_config.yml调整。访问localhost:4000即可

$ hexo server

2、github

github 是一个世界上最大的男性同胞交♂流平台（大雾）。由于github非常良心的可以免费托管开源项目，又非常良心地为那些项目设置了介绍页（Page）

所以，许多开源项目的说明文档就放在了github上面。比方说hexo.io , xxx.io 这样的网站。

因此，我们也可以把我们的技术博客放在github上面，作为技术的分享。

官方说明

https://pages.github.com/

建立项目

关于建立项目的名称。[username].github.io应该是项目名称，代表着个人主页。所以推荐用此名来命名。
如果觉得这个域名太土，不高大上的话，后文有写买域名的方法。’[username]’代表着你的github的用户名称。

上传可以选择https上传和SSH上传。若选择SSH上传需要配置好SSH keys。

SSH keys为本机和服务器(github)建立了桥梁。

生成新的SSH keys

检查本机的ssh密钥

$ cd ~/.ssh

生成keys，会让你输入两次密码。这个设置是防止别人往你的项目里提交内容。

$ ssh-keygen -t rsa -C "<email>@youremail.com"

最后看到”The keys randomart image is:”表示设置成功。

添加SSH keys到github

打开前面.ssh目录下的id_rsa.pub文件，复制所有内容，然后在github个人设置->SSH->add a public key，粘贴刚才的内容到Key文本框里，点击OK。

测试是否成功

$ ssh -T git@github.com

Hi [username]! You've successfully authenticated, but GitHub does not provide shell access.

看到这个表示设置成功。

Hexo设置git选项

如果不换域名的话。回到Hexo,在_.config.yml最后的deploy

` deploy:

type: git

repository: git@github.com:[username]/[username].github.io.git

branch: [branch]

message: [message] `

安装deploy包

$ npminstall hexo-deployer-git --save

上传Hexo内容到github

cd到Hexo项目目录，然后输入下面命令即可。

$ hexo deploy

至此，你已经可以通过[username].github.io来看到Hexo的页面了。

更换域名

买域名

想换一个狂霸酷炫吊炸天的域名？去买吧。

域名可以在godaddy买，用支付宝。其他地方也可以。

https://www.godaddy.com/

DNS解析

我们需要把你买的域名解析到你github个人主页的ip上去。

先ping一下ip

$ ping [username].github.io

然后在DNSpod的域名管理。将要解析的域名的两天记录类型为A的条目的记录值该为github个人主页的IP。
然后回到godaddy，更改 godaddy 的 Nameservers 为 DNSpod 的 NameServers:

F1G1NS1.DNSPOD.NET

F1G1NS2.DNSPOD.NET。

（似乎godaddy也可以完成解析的工作，如果这样就可以不用更改NameServers了。可能这样做有其他的好处。）

更改完之后，域名已经访问到了github，虽然并不是我们的主页，还差最后一步。

在博客项目中，新建一个文件名为CNAME。位置在项目旁边的 “+”。然后内容是我们绑定的域名，确定。

再次输入更换的域名，已经能正确显示。输入[username].github.io时，发现跳转到了更换后的域名。
